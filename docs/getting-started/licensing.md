## как заменить файл лицензии

#### Как заменить старый файл с лицензией на новый?

При возникновении ситуации, когда вам необходимо заменить ключ лицензии на другой - отдел технической поддержки, с которым вы связываетесь, отправляет вам на почту файл.

Данный файл имеет расширение \"ваш е-майл.lic\".

\- Например \"dental-clinik@gmail.com.lic\" -

![\1](../images/media/image1.png){width="6.496527777777778in" height="7.704861111111111in"}

1\. Данный файл вам необходимо скачать на свой компьютер.

!!! ВАЖНО !!! Откройте папку со скачанным файлом и проверьте, чтобы в названии файла не было скобок, а в них цифр (1), (2) и тд. !!! ВАЖНО !!!

Открыть папку со скачанным файлом можно через загрузки, например Yandex.Браузер это делается вот так:

![\1](../images/media/image2.png){width="6.496527777777778in" height="7.1375in"}

а вот так в Google Chrome:

![\1](../images/media/image3.png){width="4.84375in" height="5.416666666666667in"}

, а затем:

![\1](../images/media/image4.png){width="6.496527777777778in" height="2.796527777777778in"}

после нажатия кнопки \"Показать в папке\" откроется папка, где будет выделен синим цветом скачанный файл.

ОБЯЗАТЕЛЬНО проверьте, чтобы название файла соответствовало шаблону \"ваша е-майл.lic\".

(Если вдруг в названии файла будут скобки с цифрами внутри - то удалите все похожие файлы из этой папки и переименуйте данный фал по вышеописанному образцу \"ваша е-майл.lic\".)

![\1](../images/media/image5.png){width="6.496527777777778in" height="2.688888888888889in"}

Правой кнопкой мыши нажимаем на этот файл и копируем его.

2\. Закрыть программу iStom на компьютере.

Необходимо закрыть программу iStom перед тем как менять файл лицензии.

3\. Открываем рабочий стол и находим иконку программы iStom.

Правой кнопкой мыши нажимаем на иконку и далее нажимаем \"Расположение файла\" - должна открыться папка с файлами программы.

![\1](../images/media/image6.png){width="5.010416666666667in" height="2.9895833333333335in"}

Открылась папка с файлами программы iStom - нажимаем правой кнопкой мыши на пустое место рядом с файлами и нажимаем \"Вставить\".

![\1](../images/media/image7.png){width="6.496527777777778in" height="3.7291666666666665in"}

Так как мы меняем старый файл лицензии на новый, поэтому, когда появится окно выбора замены или пропуска - выбираем \"Заменить файл в папке назначения\"

![\1](../images/media/image8.png){width="4.677083333333333in" height="3.1875in"}

Всё, файл лицензии заменён, можно закрывать все ненужные папки.

## распр.лиц сотр

#### Распределение лицензий для сотрудников

В системе управления клиникой iStom присутствует возможность определения типа лицензии в программе для каждого пользователя.

Чтобы провести распределение лицензий между сотрудниками -- перейдите в настройки как показано на рисунке ниже.

![\1](../images/media/image1.png){width="6.496527777777778in" height="5.346527777777778in"}

Для каждого сотрудника укажите тип лицензии, которая будет за ним закреплена.

Для того, чтобы исключить сбои в работе используется алгоритм «понижения» уровня лицензии.

*Пример*

*Вы имеете лицензии:*

*Ultimate (1 шт),*

*Professional (1 шт),*

*Standart (2 шт),*

*следовательно, вы не сможете запустить более 4 активных сессий, так как работает ограничение по общему количеству активных сессий.*

*Также, в программе работает ограничение по каждому типу лицензий, а именно, если вам присвоен тип лицензии Ultimate, а в ключе количество купленных лицензий Ultimate всего одна, то следующий, кто попытается зайти с таким же типом лицензии -- будет авторизирован в программе с лицензией по входу Professional, т.е. сработает механизм понижения уровня лицензии, НО! Если вдруг и количество активных сессий по лицензии Professional будет превышать лимит -- то тип лицензии по входу определится как Standart, а если и все лицензии Standart будут «заняты» - тогда в программу вы не сможете войти. То есть: если у вас куплено 4 пакета, то одновременно в программе могут работать только 4 пользователя.*

Пользователь «Первый запуск» - также учитывается при подсчёте количества активных сессий, поэтому сверх общего ограничения запустить программу под этим пользователем не получится.

Для удобства при запросе данных для входа есть возможность посмотреть

список активных сессий, как показано на рисунке.
